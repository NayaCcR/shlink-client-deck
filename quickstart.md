# Quickstart

Link Console has two run modes:

- Static Mode: pure frontend assets. The browser stores Shlink server configs and API keys locally.
- Hosted Mode: Next.js server runtime with accounts, workspaces, encrypted Shlink credentials, protected links, and configurable storage.

## Requirements

- Node.js 20.11 or newer. Node.js 24+ is recommended for the default SQLite storage.
- npm.
- A Shlink instance URL and an API key generated by Shlink.

## Static Mode

Use this when you only need a personal browser-side Shlink console.

```bash
npm install
npm run dev
```

Open <http://localhost:3000>, then add your own Shlink API URL and API key in the UI.

Build static assets:

```bash
npm run build
```

The static output is written to `out/`.

For public demos, prefer Static Mode.

If you also run a private Hosted deployment, expose it as a separate URL and link to it from the static site:

```jsonc
// public/config.json
{
  "allowHostedMode": true,
  "hostedModeUrl": "https://links.31n.cc"
}
```

`hostedModeUrl` is the Link Console Hosted URL, not a Shlink API URL. Static assets still cannot run Hosted APIs by themselves, and this URL does not imply a public Hosted service.

## Hosted Mode

Use this when you need login, workspaces, member permissions, encrypted API keys, backend proxying, or password-protected short links.

Create a local config file:

```bash
cp link-console.config.example.json link-console.config.json
```

Windows PowerShell:

```powershell
Copy-Item link-console.config.example.json link-console.config.json
```

Edit `link-console.config.json`:

- Set `app.publicUrl` to your Link Console public URL.
- Replace `security.authSecret`.
- Replace `security.credentialEncryptionKey`.
- Keep `storage.driver` as `sqlite` for the default local database.

Start development server:

```bash
npm run dev:hosted
```

Production build and start:

```bash
npm run build:hosted
npm run start:hosted
```

Default URL: <http://localhost:3000>.

In production, put Hosted Mode behind its own domain or subdomain, then set `app.publicUrl` in `link-console.config.json` to that public URL.

## Custom Config Path

All Hosted scripts read `link-console.config.json` by default. You can use another file:

```bash
npm run dev:hosted -- --config ./config/local.json
npm run start:hosted -- --config /etc/link-console/config.json
```

Or use an environment variable:

```bash
LINK_CONSOLE_CONFIG=/etc/link-console/config.json npm run start:hosted
```

PowerShell:

```powershell
$env:LINK_CONSOLE_CONFIG="C:\link-console\config.json"
npm run start:hosted
```

## Storage

Default storage is SQLite:

```jsonc
{
  "storage": {
    "driver": "sqlite",
    "sqlite": {
      "path": "hosted-store.sqlite"
    }
  }
}
```

Relative SQLite paths are stored under `.link-console/`, so the default database file is `.link-console/hosted-store.sqlite`.

Supported drivers:

- `sqlite`: default single-node storage.
- `postgres` / `pgsql`: use `storage.databaseUrl`.
- `mysql`: use `storage.databaseUrl`.
- `redis`: use `storage.redis.url` and `storage.redis.key`.
- `json`: legacy file mode for migration/debugging.

When a new database is empty, Link Console tries to import old data from `storage.legacyJson.importPath`, usually `.link-console/hosted-store.json`.

## Protected Links

Password-protected short links are Hosted-only.

In Hosted Mode, Link Console stores the real target URL encrypted on the server. The Shlink short link points to `/r?token=...`; visitors enter the access password on that intermediate page before redirecting.

Static Mode does not support this safely because there is no trusted backend.

## Common Checks

```bash
npm run typecheck
npm run build
npm run build:hosted
```

## Notes

- Do not commit `link-console.config.json`; it is ignored by Git.
- Back up your database and `security.credentialEncryptionKey`.
- Losing the encryption key means saved Shlink API keys and protected-link targets cannot be decrypted.
- `app.publicUrl` is the Link Console panel URL, not your Shlink API URL.
